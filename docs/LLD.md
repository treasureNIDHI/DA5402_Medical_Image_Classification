# Low-Level Design (LLD) — API Endpoint Specifications

## Base URL
- Local: `http://localhost:8001`
- Docker: `http://localhost:8001`

---

## Endpoints

### 1. POST /predict

**Purpose**: Classify a medical image using a specified model.

**Request**
```
Content-Type: multipart/form-data
```

| Field | Type | Required | Values | Description |
|---|---|---|---|---|
| `file` | binary (image) | Yes | JPEG, PNG, BMP, TIFF | Medical image to classify |
| `model_type` | string (form) | Yes | `pneumonia` \| `brain` | Which model to use |

**Example**
```bash
curl -X POST http://localhost:8001/predict \
  -F "file=@chest_xray.jpeg" \
  -F "model_type=pneumonia"
```

**Response 200 OK**
```json
{
  "prediction":         "PNEUMONIA",
  "confidence":         0.9999,
  "inference_time_seconds": 0.071,
  "model_type":         "pneumonia"
}
```

| Field | Type | Description |
|---|---|---|
| `prediction` | string | Class label (`NORMAL`/`PNEUMONIA` or `glioma`/`meningioma`/`notumor`/`pituitary`) |
| `confidence` | float [0,1] | Softmax probability of predicted class |
| `inference_time_seconds` | float | Wall-clock time for the forward pass |
| `model_type` | string | Echo of the requested model |

**Special responses (low confidence or wrong modality)**
```json
{ "prediction": "No pneumonia detected (low confidence)", "confidence": 0.43 }
{ "prediction": "No tumor (input is not a brain MRI)", "confidence": 0.0 }
```

**Response 422 Unprocessable Entity** — missing/invalid form fields
```json
{ "detail": [{ "loc": ["body", "model_type"], "msg": "field required" }] }
```

**Response 500 Internal Server Error** — model not loaded
```json
{ "detail": "pneumonia model is not loaded" }
```

### 4. GET /readyz

**Purpose**: Kubernetes/Docker readiness probe. Returns non-200 if models not loaded.

**Response 200 OK** (ready)
```json
{
  "status": "ready",
  "ready":  true,
  "models_loaded": true,
  ...
}
```

**Response 503 Service Unavailable** (not ready)
```json
{ "status": "not_ready", "ready": false, "models_loaded": false }
```

## Class Labels

### Pneumonia Model (`model_type=pneumonia`)
| Index | Label | Description |
|---|---|---|
| 0 | `NORMAL` | No pneumonia detected |
| 1 | `PNEUMONIA` | Pneumonia detected |

### Brain Tumor Model (`model_type=brain`)
| Index | Label | Description |
|---|---|---|
| 0 | `glioma` | Glioma tumor |
| 1 | `meningioma` | Meningioma tumor |
| 2 | `notumor` | No tumor present |
| 3 | `pituitary` | Pituitary tumor |

## Error Codes Summary

| HTTP Code | Meaning | When |
|---|---|---|
| 200 | OK | Successful request |
| 422 | Unprocessable Entity | Missing/invalid form fields |
| 500 | Internal Server Error | Model not loaded, unexpected exception |
| 503 | Service Unavailable | Readiness probe when models not ready |

---

## Input Validation Rules

1. `file` must be a valid image file (JPEG, PNG, BMP, TIFF)
2. `model_type` must be exactly `"pneumonia"` or `"brain"` (case-sensitive)
3. Image is preprocessed server-side: resized to 224×224, converted to RGB
4. Modality detection rejects mismatched images (X-ray sent to brain model, or vice versa) returning confidence 0.0
5. If model confidence < 0.7, a "low confidence" message is returned instead of a class label
